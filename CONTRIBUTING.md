# 贡献指南

感谢您对本项目的关注！我们欢迎各种形式的贡献。

## 📋 贡献方式

### 1. 报告问题
- 在 [Issues](https://github.com/xiaodaipi173/autoUpdateSSL/issues) 中提交 Bug 报告
- 提供详细的复现步骤和环境信息
- 附上相关的日志和错误信息

### 2. 提交功能建议
- 在 Issues 中提交功能请求
- 详细描述功能需求和使用场景
- 说明该功能如何改进项目

### 3. 提交代码
1. Fork 本仓库
2. 创建您的特性分支：`git checkout -b feature/AmazingFeature`
3. 提交您的更改：`git commit -m 'Add some AmazingFeature'`
4. 推送到分支：`git push origin feature/AmazingFeature`
5. 开启一个 Pull Request

## 📝 代码规范

### 代码风格
- 使用 2 空格缩进
- 使用有意义的变量和函数名
- 添加必要的注释（特别是复杂逻辑）
- 保持代码简洁易读

### 提交信息规范
使用清晰的提交信息，格式如下：
```
类型: 简短描述

详细描述（可选）
```

类型包括：
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建/工具链相关

示例：
```
feat: 添加证书自动续期功能

- 实现证书有效期检查
- 添加自动续期逻辑
- 更新相关文档
```

